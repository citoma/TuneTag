# 乐签（TuneTag）一期产品需求文档（PRD）

## 1. 产品概述

- 产品名称：乐签
- 英文名称：TuneTag
- 平台：macOS（本地应用）
- 产品定位：
  一款聚焦“本地媒体标签编辑”的轻量工具。用户通过批量拖拽导入媒体文件，在列表中完成单个或批量标签修改，并写回文件。

### 1.1 一期核心价值

TuneTag V1 只解决一件事：

**把一批媒体文件拖进来，然后快速改它们的标签信息。**

不做媒体管理器，不做播放器，不做复杂内容消费体验。

---

## 2. 一期最小闭环（MVP）

用户完整路径：

1. 拖入文件
2. 列表展示
3. 选中单个或多个文件
4. 编辑属性
5. 保存写入

该路径构成 V1 的全部核心闭环。

---

## 3. 功能范围（In Scope）

### 3.1 导入

主入口：**批量拖拽导入**。

支持拖入：
- 单个文件
- 多个文件
- 文件夹（自动递归解析可识别媒体文件）

可保留辅助按钮：`选择文件`（非主交互路径）。

#### 导入结果要求

- 自动解析媒体文件并生成列表
- 过滤不支持格式并给出提示（例如“3 个文件格式不支持，已跳过”）
- 去重策略：同路径文件仅保留一条记录

### 3.2 文件列表展示

导入后进入主工作区，列表为主视图。

建议字段：
- 文件名
- 格式
- 标题
- 艺术家
- 专辑
- 年份
- 状态（未修改 / 已修改 / 保存失败）

支持能力：
- 多选
- 全选
- 搜索
- 排序

### 3.3 单文件属性编辑

当仅选中 1 个文件时，右侧显示单文件编辑面板。

V1 可编辑字段（命名建议：`基础标签` 或 `基础信息`）：
- 标题
- 艺术家
- 专辑
- 年份
- 备注
- 曲目号

兼容性要求：
- 若文件格式不支持某字段：
  - 显示“该格式不支持此字段编辑”，或
  - 直接隐藏该字段（二选一，保持全局一致）

### 3.4 批量修改

当选中多个文件时，右侧切换为批量编辑模式。

V1 支持 4 类操作：

1. 批量设值
- 为某字段统一设置同一值
- 示例：`专辑 = 春风合集`、`年份 = 2026`

2. 批量替换
- 在某字段中执行查找/替换
- 示例：标题中 `demo` 替换为 `正式版`

3. 批量清空
- 清空指定字段

4. 批量追加
- 增加前缀或后缀
- 示例：标题前缀 `Live -`；艺术家后缀 ` feat.`

### 3.5 保存写入

必须具备：
- 保存按钮
- 写入进度
- 成功/失败反馈
- 失败文件提示
- 未保存修改提示

保存完成示例反馈：
- `已成功保存 28 个文件`
- `失败 2 个，请查看详情`

---

## 4. 非目标（Out of Scope）

一期明确不做：
- Web 端
- Windows 端
- 在线播放
- 封面编辑
- 歌词编辑
- 自动抓取元数据
- 文件重命名规则
- 文件整理归档
- 账号系统
- 云同步
- 复杂媒体库管理

---

## 5. 信息架构与页面结构

V1 建议维持“单主界面”结构。

### 5.1 空状态页

中心区域展示：
- 拖拽区域
- 提示文案：`拖入媒体文件开始编辑`
- 可选辅助按钮：`选择文件`

### 5.2 主工作区

- 左/中：文件列表
- 右侧：属性编辑区
  - 单选：单文件基础标签
  - 多选：批量编辑面板
- 底部：操作区
  - 保存
  - 取消修改
  - 修改数量提示

不引入多页面跳转，降低操作复杂度。

---

## 6. 关键交互原则

1. 首屏即拖拽
- 打开即可理解主操作，减少学习成本。

2. 列表是主角
- 围绕批量文件处理展开布局。

3. 编辑要直接
- 点击字段即改，不进入二级页。

4. 批量模式明显
- 明确提示：`已选中 N 个文件，当前为批量编辑模式`。

5. 保存状态清晰
- 未保存、已保存、保存失败状态可快速区分。

---

## 7. 状态模型（建议）

### 7.1 文件级状态
- `clean`：未修改
- `dirty`：已修改待保存
- `saving`：正在写入
- `saved`：已写入成功（可在下一次编辑后回到 dirty）
- `error`：写入失败

### 7.2 页面级状态
- 空状态（无文件）
- 已导入（有文件，单选）
- 已导入（有文件，多选）
- 保存中（全局进度）

---

## 8. 异常与提示（建议最小集）

- 拖入空文件夹：提示“未发现可编辑媒体文件”
- 拖入不支持格式：提示跳过数量
- 保存时文件被占用/无权限：标记失败并给出原因
- 关闭窗口前存在未保存修改：二次确认

---

## 9. 一期验收标准（Acceptance Criteria）

1. 导入
- 能通过拖拽导入单文件、多文件、文件夹
- 导入后正确展示列表

2. 列表
- 支持多选、全选、搜索、排序
- 状态列能正确反映 clean/dirty/error

3. 编辑
- 单选时可编辑基础标签字段
- 多选时可执行四类批量操作

4. 保存
- 执行保存后可见进度
- 成功/失败数量反馈准确
- 失败文件可识别并可重试

5. 体验
- 用户可在 1 分钟内完成“拖入 20+ 文件并批量改专辑后保存”的任务

---

## 10. 版本边界一句话

**TuneTag V1 = macOS 本地、拖拽导入、列表驱动、单改+批改基础标签、可见可控地保存写回。**
